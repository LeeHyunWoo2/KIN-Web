## 📢 전하고 싶은 말

제가 단기간에 어디까지 해낼 수 있는지 **적응력과 발전 속도를** 증명해보고 싶었고, 그래서 이 프로젝트에 착수했습니다.

- **약 70일** 동안 익숙하지 않은 새로운 기술 스택을 학습하며,
- **설계, 성능, 보안, UX, 유지보수를 고민하고 적용하려 노력**했습니다.
- 각 인프라의 상호작용을 이해하고, **데이터 흐름을 파악하며 학습**했으며,
- 어떤 환경에서도 **문제를 해결하고 적응하려는 자세로 임했습니다.**
- **품질과 안전성을 높이기 위해 지속적으로 고민하며 발전**할 수 있었습니다.

불과 **8개월 전**만 해도, Java를 처음 배우는 **초심자**에 지나지 않았습니다.<br/>
그때 [학원에서 생전 처음 작성해본 코드가 기억이 나네요.](https://github.com/LeeHyunWoo2/JAVA/blob/ba6772ad85a6cac6b99827ea15a7c80efc1798f8/ch01/Hello.java#L11) `System.out.println("자바지옥에 오신것을 환영합니다.");{:java}`<br/>

하지만 지금은 이만큼 성장하여, **혼자서 프로젝트의 설계, 구현, 배포까지 모든 과정을 주도**할 실력을 갖추게 되었습니다.

## 🎯 프로젝트의 본질

이 앱은 **제 가능성을 시험하는 데에 의미를 둔** 프로젝트 입니다.

개발자로서, 새로운 기술을 얼마나 빠르게 학습하고, 이를 실전에 적용할 수 있는지를 보여주고자 했습니다.

**그래서:**<br/>
✅ 프론트엔드, 백엔드, 데이터베이스, 배포까지 **바닥에서부터 모든 것을 혼자 설계하고 구현**했습니다.<br/>
✅ 단순한 기능 나열이 아니라, **보안, 성능, UX, 유지보수까지 고민하며 설계**했습니다. 덕분에 트레이드오프 해결 경험이 꽤 있습니다.<br/>
✅ 특정 기술이나 프로그래밍 언어에 국한되지 않고, **어떤 환경에서도 빠르게 적응할 수 있는 역량을 강조**하려 합니다.<br/>
✅ 더불어, **모든 인프라가 무료플랜이라는 제약아닌 제약** 때문에, 성능 최적화를 위해 많이 신경썼습니다.

### 기능 구현에 그치지 않고

- "**이 문제를 가장 효과적으로 해결할 방법은 무엇인가?**"
- "**이 결정이 앞으로 어떤 영향을 미칠 수 있을까?**" <s>(이런식으로 만들면, 고치면 나중에 무슨 사고가 날 수 있는가?)</s>
- "**어떻게 하면 성능을 더 쾌적하게 개선할 수 있을까?**"

와 같은 질문을 지속적으로 던지며 학습했습니다.<br/>

이 프로젝트를 통해<br/>

**보안** - 사용자 데이터 보호를 위한 인증, 쿠키 보안, CORS, 세션 관리<br/>
**성능** - 데이터 압축, 캐싱, API 최적화, 서버 부하 감소<br/>
**UX** - UI 디자인뿐만 아니라, 백엔드 설계, API 응답 속도, 데이터 구조, 인증 방식까지 고려한 사용자 경험<br/>
**유지보수** - 확장 가능한 코드 구조와 명확한 설계 패턴<br/>

등 다양한 측면을 고민하며, 단순히 작동하는 프로그램이 아닌 **품질 높은 애플리케이션**을 만들기 위해 고민했습니다.

라이브러리를 선택할 때에도 "이게 좋다더라"라는 글만 찾아보며 의존하지 않고,<br/>
후보를 정리한 후 Github와 공식 문서를 비교 검토하며 **현 상황에 가장 적합한 선택**을 하기위해 노력했습니다.<br/>
덕분에 API 문서를 효과적으로 활용하는 데 익숙해졌으며, 이는 개발자로서 가장 중요한 역량 중 하나가 아닐까 생각합니다.<br/>
