## 🎯 전하고 싶은 말

저는 전통적인 컴퓨터 공학 교육을 받지 않았고, KDT라는 열린 교육과정을 통해 개발을 시작했습니다.<br/>
하지만 저는 단순히 주어진 강의만을 따라간 것이 아니라, **처음부터 끝까지 책임지고** 경험하는 실전형 프로젝트를 구현하고 싶었고,<br/>
**실제 운영 가능한 서비스를 웹상에 배포하고, 인프라를 직접 다루며 발생하는 문제를 직접 해결하는 과정을 겪으며** 성장해 왔습니다.  <br/>

불과 **8개월 전**만 해도, Java를 처음 배우는 **초심자**에 지나지 않았습니다.<br/>
[학원에서 생전 처음 작성해본 코드가 기억이 나네요.](https://github.com/LeeHyunWoo2/JAVA/blob/ba6772ad85a6cac6b99827ea15a7c80efc1798f8/ch01/Hello.java#L11) `System.out.println("자바지옥에 오신것을 환영합니다.");{:java}`<br/>

<br/>
이 프로젝트를 통해 제가 개발자로서 새로운 기술을 얼마나 빠르게 학습하고 실전에 적용할 수 있는지 충분히 전달되기를 바랍니다.

### 기능 구현에 그치지 않고

- "**현재 증상이 어떤 문제로 발생한 것 인지 정의할 수 있는가?**"
- "**이 문제를 가장 효과적으로 해결할 방법은 무엇인가?**"
- "**이 결정이 앞으로 어떤 영향을 미칠 수 있을까?**" <s>(이런식으로 만들면, 고치면 나중에 무슨 사고가 날 수 있는가?)</s>
- "**어떻게 하면 성능을 더 쾌적하게 개선할 수 있을까?**"

와 같은 질문을 지속적으로 던지며, 실전으로 문제를 직면하고<br/>

라이브러리를 선택할 때에도 "이게 좋다더라"라는 글만 찾아보며 의존하지 않고,<br/>
후보를 정리한 후 Github와 공식 문서를 비교 검토하며 **현 상황에 가장 적합한 선택**을 하기위해 노력했습니다.<br/>
